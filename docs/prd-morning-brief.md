# PRD — Morning Brief (live slice)

**Status:** shipped (PR #11)
**Slice:** next in the dashboard-live effort (first Research-tab panel to go live)
**Date:** 2026-06-21

## Goal

Replace the static **Morning Brief** panels — on both the dashboard brief card and the
full Morning Brief view — with live data, generated by a headless `/morning-brief` slash
command and painted from today's daily-note frontmatter. v1 covers three of the four
panels; **𝕏 Conversation is deferred** to a later slice.

## Panels & sources (decided)

| Panel | v1 source |
|-------|-----------|
| Headlines | Web search (broad AI/tech) **+** Hacker News top stories, merged & deduped to ~5 |
| Reading Queue | Obsidian notes — a configured read-later tag/folder |
| Note Opportunities | Vault analysis — topic gaps, atomize candidates, MOC links |
| 𝕏 Conversation | **Deferred** — panel shows an empty/placeholder state; a later slice picks API vs web-search vs manual |

## Success criteria

- Run **Morning Brief** (card ⟳ or the Quick Action) → today's daily note gets a `brief:`
  key → the three panels repaint with real data (dashboard card **and** full view) on the
  next refresh.
- The header counts (Headlines N / Reading N / Notes N) reflect the live array lengths.
- No `brief:` key → the three panels show a graceful empty state (`▶ RUN MORNING BRIEF`);
  the 𝕏 panel shows its deferred state. Never a crash or stale placeholder.
- `pnpm run build` (the only correctness gate — `tsc -noEmit` + bundle) passes.

## Scope

**In:** the `/morning-brief` slash command (writes `brief:`); a `brief.ts` read of the
frontmatter into the three panels (card + full view); empty states; header counts; the
card ⟳ / Quick Action trigger.

**Out (later slices):** 𝕏 Conversation data; the full toolbar's search/filter/sort and
section chips being interactive; auto-running the brief on a schedule; the sibling
Release Radar & Hacker News panels.

## Source model — command writes, plugin paints (decided)

Mirror the Schedule + Tasks slice (PR #7). A **project-scoped** `/morning-brief` command
in the vault's `.claude/commands/`, invoked by the card ⟳ button / "Morning Brief" Quick
Action as `/Users/jay/.local/bin/claude -p "/morning-brief"` with cwd = vault root. The
command does the research + synthesis (it *is* the LLM — it can web-search, read the
vault, and summarize); the plugin only reads the frontmatter it leaves behind. Absolute
`claude` path because GUI-launched Obsidian has a minimal PATH.

## Where the brief lives (decided — flag to sanity-check)

Today's **daily note** frontmatter, under a `brief:` key — the same note and path
resolution as `schedule:`/`tasks:` (`daily-notes/<YYYY-MM-DD>.md`, honoring the core
Daily Notes config). Rationale: it is "today's brief," one note per day, and it reuses
`dayplan.ts`'s date/path logic plus the existing `metadataCache` "changed" watcher.
(Alternative — a dedicated `brief/<YYYY-MM-DD>.md` — rejected for v1 to avoid a second
note-path convention; revisit if briefs need to grow beyond what frontmatter holds well.)

## Data contract — `brief:` frontmatter

File: `daily-notes/<YYYY-MM-DD>.md`

```yaml
---
brief:
  generated: "2026-06-21T07:32"        # optional; shown as the brief date/time
  headlines:
    - { text: "Anthropic ships an extended context window for agentic coding.", src: "anthropic.com" }
  reading:
    - { title: "The Hidden Cost of Microservices", src: "martinfowler.com", meta: "14 min read" }
  notes:
    - "No note yet on *local-first sync* — three sources cited it this week."
---
```

- `headlines[]`: `{ text, src? }` → `.brief__bullet`. Card shows ~3, the full view shows
  all (~5). `src` optional, surfaced in the full view.
- `reading[]`: `{ title, src, meta }` → `.reading-item__title` / `__src` / `__time`.
  `meta` is free text ("14 min read", "saved 2d ago", "in progress").
- `notes[]`: `string[]` (may contain markdown `*em*`) → the numbered `.opp-item` rows.
- All keys optional; an absent key → that panel's empty state. No `x:` key is written in v1.

## Command behavior — `/morning-brief`

- **headlines** ← web search across AI/tech **and** Hacker News top stories; dedupe and
  rank to ~5 one-line summaries, each with a source domain.
- **reading** ← notes carrying the configured read-later tag/folder (leaning default: tag
  `#to-read`), newest/unread first, ~6, each with title, source/domain, and a short meta.
- **notes** ← scan recent captures/readings for: topics cited repeatedly with no dedicated
  note, capture notes worth atomizing, and recent readings to link into an existing MOC;
  ~4 suggestions.
- Writes/merges `brief:` into **today's** daily note, preserving the body and any other
  frontmatter (`schedule`/`tasks`). Idempotent — re-running re-snapshots in place.
- v1: does **not** write `x:`.

## Plugin behavior

- `brief.ts` — `readBrief(app): Brief` parses today's daily-note `brief:` from the
  in-memory metadata cache (synchronous, like `dayplan.ts`; no stale-paint token needed).
- `refreshBrief()` / `paintBrief()` paint the brief panels within `.agentic-os`, in
  whichever view is rendered (dashboard card or `full-brief` — they share the
  `.brief__panel` classes). The card truncates headlines to ~3; the full view shows all.
  Header counts come from the array lengths.
- Repaint cadence: extend the existing daily-note `metadataCache.on("changed")` watcher
  (added for `dayplan`) to also call `refreshBrief()`, plus the 60s tick and the
  active-leaf-change repaint.
- The card ⟳ button and the "Morning Brief" Quick Action trigger generation via the Shell
  Commands command; the panels repaint from disk when the command finishes and the note
  changes (the watcher fires).
- Degrade: no note / no `brief:` / parse error → per-panel empty state; the 𝕏 panel shows
  its deferred state.

## Settings

- `briefReadingTag` (leaning default `#to-read`) and/or `briefReadingFolder` — the Reading
  Queue source (resolve which at authoring).
- Reuses the daily-note path resolution and the `claude` invocation conventions from the
  Schedule slice; no new path config.

## Verification

1. Author `/morning-brief`; run `claude -p "/morning-brief"` in the vault → inspect the
   daily note: valid `brief:` (`headlines`/`reading`/`notes`), body + `schedule`/`tasks`
   untouched.
2. `pnpm run build` passes.
3. In Obsidian: the card and the full view paint the three live panels; header counts
   match; the 𝕏 panel shows the deferred state.
4. Delete `brief:` → the three panels fall back to the empty state cleanly.

## Resolved decisions

1. **v1 panels** — Headlines, Reading Queue, Note Opportunities go live; **𝕏 Conversation
   is deferred** (X data-access constraints). *(Reconciles the scope answer, which included
   𝕏, with the source answer, which deferred it: a panel can't ship live with no source.)*
2. **Headlines source** — web search **and** Hacker News, merged.
3. **Reading Queue source** — Obsidian notes (a read-later tag/folder).
4. **Source model** — a headless `/morning-brief` writes `brief:`; the plugin reads it,
   mirroring Schedule + Tasks.

## Remaining to confirm at authoring

- Reading-source selector: tag vs folder, and the default value.
- Note-opportunities heuristic: how aggressive, and which folders count as "captures".
- 𝕏 deferred-panel treatment: keep the static designer placeholder vs a quiet
  "coming soon" empty state (and excluded from the header voice count).
- Daily-note `brief:` key vs a dedicated brief note (leaning daily note — flagged above).

## Dependency note

The plugin can only paint what's on disk, so the order within this slice is: author
`/morning-brief` (writes `brief:`) → wire the `brief.ts` read → paint. It reuses the
daily-note path + `metadataCache` watcher from the Schedule slice (already shipped), so
no Daily Notes reconfiguration is needed.

## 𝕏 Conversation — deferred slice (notes for later)

Options when this is picked up: the official **X/Twitter API** (paid tier, token kept in
the command's environment); a **web search** for what key voices are saying (no API, but
approximate/lossy); or a **manually-seeded** handle/take list in a note that the command
formats. Until then the panel renders an empty/placeholder state and is excluded from the
header's "𝕏 Voices" count.
